# Тренер Алины — правила проекта

Фитнес-трекер Алины: дневник питания, тренировки, вес, цель. Разговаривай с Алиной по-русски,
простыми словами — она не программист.

## Архитектура (важно понять перед любой правкой)

Приложение — **один самодостаточный HTML-файл**, без сборщиков, без npm, без фреймворков.
Никакого Expo/React Native тут больше нет (мигрировали с него в 2026-07).

- **`docs/index.html`** — всё приложение: вся разметка, все стили, вся логика в одном файле
  (HTML + инлайн `<style>` + инлайн `<script>` на чистом vanilla JS). Именно этот файл вы
  правите, когда меняете что угодно в UI/логике.
- **`docs/data/*.json`** — база продуктов (food-curated / food-calorizator / food-generated),
  вынесена отдельно от `index.html`, потому что вместе это ~1 МБ. Подгружается лениво через
  `fetch()` при первом открытии поиска еды. Трогать эти файлы нужно редко — это справочные
  данные, не код.
- **GitHub Pages** раздаёт папку `docs/` как обычный сайт: `https://alina3500-2-arch.github.io/trener-aliny/`.
  Приложение внутри iOS-обёртки просто открывает этот URL в WKWebView.
- **`ios-native/`** — тонкая нативная iOS-обёртка (Swift, чистый UIKit, без Storyboard,
  без CocoaPods/SPM). Всё, что она делает: (1) грузит вышеуказанный URL на весь экран,
  (2) разрешает доступ к камере/микрофону для WKWebView, (3) мостит локальные уведомления
  через `window.webkit.messageHandlers.notify`. Больше в ней ничего нет и не должно быть.
- **`groq-proxy/`** — отдельный маленький Cloudflare Worker (не часть iOS/веб-приложения).
  Прокси к Groq API (обходит гео-блокировку 403). Уже задеплоен на
  `https://divine-dream-5ce0.alinatyu456.workers.dev` — трогать не нужно, если Groq не начнёт
  давать сбои сам по себе.
- **`.github/workflows/ios-build.yml`** — CI-сборка неподписанного `.ipa` для SideStore.
  Срабатывает **только** когда меняется `ios-native/**` (или сам workflow-файл) — обычные
  правки `docs/index.html` его НЕ запускают и пересборки не требуют.

## Главное правило: как вносить изменения

**99% задач — это правка одного файла `docs/index.html`.**

1. Открой `docs/index.html`, найди нужную секцию (внутри файла есть комментарии-разделители
   вроде `/* ===== TODAY SCREEN ===== */`, `/* ===== WORKOUT SCREEN ===== */` и т.д.).
2. Поправь.
3. Проверь в браузере (см. «Как тестировать» ниже).
4. Закоммить и запушь в `main` (или в ветку, если работаешь через PR).
5. Готово — GitHub Pages подхватит новую версию за ~минуту, приложение на телефоне обновится
   само при следующем открытии (нативная обёртка просто перезагружает страницу). **Пересборка
   и переустановка `.ipa` для этого НЕ нужна.**

Пересборка `.ipa` нужна, только если меняется сама папка `ios-native/` (например, добавляется
новый нативный мостик) — это редкий случай.

## Что нельзя ломать

- **Ключ localStorage `trener_aliny_state_v1`** и форма объекта состояния (`emptyState` в
  начале `<script>`). Все данные Алины (вес, тренировки, дневник питания, цель) лежат в этом
  одном JSON внутри localStorage браузера/WKWebView. Если поменять ключ или несовместимо
  изменить форму состояния без миграции — данные Алины "исчезнут" (на самом деле останутся
  под старым ключом, но приложение их не увидит). Если нужно поменять форму state — пиши
  миграцию в `loadState()` (там уже есть `Object.assign(emptyState, parsed)` — новые поля
  подставляются дефолтами автоматически, старые поля не удаляются просто так).
- **Формат вызова нативного моста уведомлений**: `window.webkit.messageHandlers.notify
  .postMessage({reminders:[...], workout:{days:[...], time:'HH:MM'}|null})`. Меняешь формат в
  `docs/index.html` (функция `scheduleReminders`) — обязательно синхронно поменяй разбор в
  `ios-native/TrenerAliny/WebViewController.swift` (там же). Иначе уведомления молча перестанут
  работать (мост просто ничего не поймёт).
- **`GROQ_PROXY` URL** (`https://divine-dream-5ce0.alinatyu456.workers.dev` в `docs/index.html`)
  — должен совпадать с реальным адресом Cloudflare Worker из `groq-proxy/`.
- **`appURL` в `ios-native/TrenerAliny/WebViewController.swift`** — должен указывать на
  реальный GitHub Pages адрес. Если он разъедется с URL, на котором реально раздаётся
  `docs/`, — обёртка будет грузить не то (или белый экран).
- Промпты для ИИ (Groq) в секции `/* ===== AI (Groq via Cloudflare Worker proxy) ===== */` —
  они тщательно откалиброваны (например, «порция должна заполнять 60–100% остатка калорий»).
  Меняй с осторожностью и тестируй на реальных сценариях, а не только «звучит логично».

## Как тестировать

UI-правки — **всегда** проверяй в браузере перед тем как считать готовым:

```bash
cd docs && python3 -m http.server 8080
```

Открой `http://localhost:8080/index.html`, пройди по всем 5 вкладкам (Питание, Тренировки,
Цель, Вес, История), открой консоль браузера (`F12`) и убедись, что нет красных ошибок.
Проверь конкретный сценарий, который менял (например, если правил дневник питания — добавь
через «Вручную» тестовый продукт и убедись, что калории/кольцо пересчитались).

Учти: без реального интернет-доступа к `GROQ_PROXY` вызовы ИИ (голос/фото/советы/генерация
программ) будут падать с ошибкой — это ожидаемо в песочнице, приложение в этом случае тихо
откатывается на локальные (не-ИИ) подсказки. На реальном телефоне с интернетом ИИ-функции
заработают.

Для проверки нативного моста уведомлений браузер не подходит (в нём просто `console.log`
вместо реального уведомления) — это можно проверить только на реальном iPhone после установки
`.ipa`.

## Как собрать и поставить `.ipa` (нужно редко)

Только если менял `ios-native/`:

1. На GitHub: вкладка **Actions** → workflow **iOS WebView wrapper (unsigned IPA)** →
   **Run workflow** (или просто запушь изменения в `ios-native/**` — сборка запустится сама).
2. Через несколько минут в результатах будет `.ipa` — доступен как Actions-artifact, как
   ассет GitHub Release, и в ветке `ipa` (для скачивания через `git fetch`).
3. Установи через SideStore на телефоне (как раньше).

## GitHub Pages — разовая настройка

Если сайт ещё не открывается по `https://alina3500-2-arch.github.io/trener-aliny/`, зайди в
**Settings → Pages** репозитория и включи: **Source: Deploy from a branch**, ветка — та, где
лежит актуальный `docs/` (после мержа в `main` — выбери `main`), папка — **`/docs`**. Сохрани.
Через минуту-две сайт станет доступен. Дальше ничего настраивать не нужно — каждый пуш в эту
ветку с изменением `docs/` обновляет сайт автоматически.

## Структура репозитория (после миграции)

```
docs/
  index.html          — всё приложение (правишь ты в 99% случаев)
  data/
    food-curated.json       — база продуктов (ручные, ~345)
    food-calorizator.json   — база продуктов (Calorizator.ru, ~2436)
    food-generated.json     — база продуктов (Open Food Facts, ~6500)
ios-native/
  TrenerAliny.xcodeproj/
  TrenerAliny/
    AppDelegate.swift
    WebViewController.swift   — вся нативная логика (WKWebView + мост уведомлений)
    Info.plist
    Assets.xcassets/
groq-proxy/                — Cloudflare Worker (ИИ-прокси), отдельный от приложения
.github/workflows/
  ios-build.yml           — CI сборки .ipa, триггерится только на ios-native/**
CLAUDE.md                 — этот файл
```

## Данные и приватность

Все данные Алины (вес, тренировки, дневник питания) хранятся **только локально** на её
устройстве (localStorage внутри WKWebView) — ничего не уходит на сервер, кроме опциональных
запросов к Groq (текст еды/фото/голос для распознавания — не хранится на стороне прокси).
Никогда не добавляй синхронизацию/бэкенд для этих данных без явной просьбы — это осознанный
архитектурный выбор (простота и приватность).
